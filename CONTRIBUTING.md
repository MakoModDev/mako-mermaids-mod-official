# Contributing to Mako Mermaids Mod

First off, thank you for considering contributing to the Mako Mermaids Mod! It's people like you that make the modding community such an amazing place to learn, inspire, and create.

Following these guidelines helps to communicate that you respect the time of the developers managing and developing this open source project. In return, they should reciprocate that respect in addressing your issue, assessing changes, and helping you finalize your pull requests.

### 1. Reporting Bugs
This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

* **Check existing issues.** Before creating a new issue, please check if the bug has already been reported.
* **Use the Template.** When you create a new issue, select the **Bug Report** template.
* **Be specific.** Include the Minecraft version (e.g., 1.21), Mod Loader version (NeoForge), and steps to reproduce the crash/bug.
* **Attach Logs.** If the game crashed, please provide a crash report or `latest.log` (via a pastebin link, not directly pasted in the text).

### 2. Suggesting Enhancements
This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.

* **Use the Template.** Select the **Feature Request** template when creating an issue.
* **Describe the Goal.** Explain *why* this feature would be useful. "I want X because Y."
* **Context.** Screenshots or mockups are always helpful!

### 3. Pull Requests (Code Contributions)
We welcome pull requests! Whether you want to fix a typo, fix a bug, or implement a new feature.

1.  **Fork the repository** and create your branch from `main`.
2.  **Setup the Workspace.** Ensure you can run the mod in your IDE (IntelliJ IDEA recommended). Run standard NeoForge setup tasks (e.g., `./gradlew genIntellijRuns`).
3.  **Test your changes.** Make sure the game builds and runs without crashing.
4.  **Follow the Style.** Try to keep your code style consistent with the existing project structure.
5.  **Submit the PR.** Provide a clear description of what you changed and link any relevant issues (e.g., "Fixes #123").

## üé® Asset Contributions (Textures/Models)
If you want to contribute textures or models:
* Please ensure you have the rights to the assets (no stolen textures).
* Match the style of the mod.
* You can submit them via a Pull Request (placing them in `assets/makomermaidsmod/textures/...`) or share them on our [Discord Server](https://discord.com/invite/8N774c8Ujd).
